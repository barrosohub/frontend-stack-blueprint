# Internationalization (i18n)

## Format.js + react-intl

| Attribute | Value                                                         |
| --------- | ------------------------------------------------------------- |
| Role      | Runtime formatting (numbers, dates, plurals) + React bindings |
| Status    | ✅ Core                                                       |
| Install   | `npm install react-intl`                                      |

### Why Format.js

- ICU MessageFormat standard — industry-proven
- Handles plurals, gender, selects natively
- Number and date formatting per locale
- Extraction tooling for translation workflows
- React bindings via react-intl

### Setup

```typescript
import { IntlProvider } from 'react-intl';

const messages = {
  'greeting': 'Hello, {name}!',
  'items.count': '{count, plural, =0 {No items} one {# item} other {# items}}',
};

function App(): JSX.Element {
  return (
    <IntlProvider locale="en" messages={messages}>
      <MyApp />
    </IntlProvider>
  );
}
```

### Usage Patterns

```typescript
import { useIntl, FormattedMessage } from 'react-intl';

// Component-based
function Greeting({ name }: { name: string }): JSX.Element {
  return <FormattedMessage id="greeting" values={{ name }} />;
}

// Hook-based (for attributes, titles, etc.)
function SearchInput(): JSX.Element {
  const intl = useIntl();
  return (
    <input
      placeholder={intl.formatMessage({ id: 'search.placeholder' })}
    />
  );
}

// Number formatting
function Price({ amount }: { amount: number }): JSX.Element {
  const intl = useIntl();
  return <span>{intl.formatNumber(amount, { style: 'currency', currency: 'USD' })}</span>;
}
```

### Rules

- Use `FormattedMessage` for visible text in JSX
- Use `intl.formatMessage()` for attributes (placeholder, aria-label, title)
- Use `intl.formatNumber()` for numbers and currencies
- Use `intl.formatDate()` for dates (uses native `Intl.DateTimeFormat`, not date-fns)
- Message IDs must be descriptive: `'user.profile.title'` not `'title1'`
- Keep translation files per locale: `src/i18n/en.json`, `src/i18n/pt.json`
